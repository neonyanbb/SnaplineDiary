В предыдущей статье мы разобрали, зачем фиксировать информацию. Здесь задача практическая: как создать новую запись от входа до сохранения.

В Snapline Diary процесс создания уже простой. Этот материал просто показывает полный маршрут шаг за шагом.

1. Два входа для создания записи

Начать можно двумя способами:

  1. Кнопка + на главной: удобно для осознанной сортировки материалов (например, фото с лекции или поездки).
  2. Системное меню «Поделиться»: удобно, когда нужно сохранить контент прямо из другой app.

Пути разные, результат одинаковый: создается запись и сразу попадает в таймлайн на главной.

Подсказка: Share Extension сейчас принимает только изображения. Если в выборе есть видео, Snapline Diary может не появиться в панели шаринга. Оставьте только изображения и поделитесь повторно.

2. Внутри экрана: пять шагов

1) Добавить фото

Обычно быстрее сначала добавить фото, потом писать заметку. В одной записи можно добавлять фото поэтапно, а лента миниатюр помогает сразу проверить состав материалов.

2) Выбрать обложку (перетаскиванием)

Лента изображений поддерживает drag-and-drop. Первая картинка становится обложкой.

Если фото несколько, поставьте самое показательное первым — запись легче узнавать на главной.

3) Написать заметку (поддерживаемый формат)

  • Многострочный обычный текст;
  • Поддержка emoji (например 🎯 и ✈️);
  • Вставленные ссылки распознаются и открываются из карточки детали;
  • В этом редакторе нет Markdown-разметки;
  • Теги хранятся отдельно от основного текста.

Фокус на надежности: зафиксировать, пересмотреть, вернуться к источнику.

4) Добавить теги (с рекомендациями)

Есть два способа:

  • Ввод вручную (пробел/enter для последовательного добавления);
  • Нажатие на рекомендованные теги.

Подсказка: символ # не нужен. Введите слово тега и нажмите пробел или enter — появится цветной тег-чип.

Рекомендации формируются по словам из текущей заметки и по вашей истории тегов.

5) Быстрые кнопки (Тег / Вставить)

  • Тег: сразу переводит фокус в поле тегов;
  • Вставить: читает буфер обмена и вставляет в заметку.

Эти кнопки убирают лишние действия в быстром режиме сохранения.

3. Что происходит после сохранения

После сохранения запись пишется локально и сразу видна на главной (обычно в верхней части сегодняшней группы).

Записи из шаринга и из кнопки + объединяются в один таймлайн.

4. Реальные сценарии

Сценарий A: быстро сохранить полезный видеоконтент

  1. Сделать скриншоты в видео-приложении;
  2. Поделиться в Snapline Diary;
  3. Вставить тезисы/транскрипт и ссылку на источник;
  4. Добавить 2-4 тега;
  5. Сохранить и потом найти по времени или по тегам.

Так «я посмотрел» превращается в «я сохранил с доказательством».

Сценарий A: создание записи через Share Extension

Сценарий B: тематическая организация с главной

  • Собрать фото ключевых моментов лекции в одну запись;
  • Собрать тематические фото поездки в одну запись;
  • Поставить репрезентативную обложку, добавить вывод и теги.

Дальше можно пересматривать либо по времени на главной, либо по темам через теги.

5. Итог

Суть не в «еще одном экране ввода», а в стабильном процессе:

Захват через два входа → выбор фото и обложки → заметка и ссылки → теги и рекомендации → сохранение в таймлайн.
