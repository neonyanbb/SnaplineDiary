Pro 备份里，一条节点长什么样：用 metadata 对照界面

系列 · 其三 · 功能篇 · 备份与节点数据 · 约 15 分钟 · 2026-04-11

本篇聚焦数据层：当你导出 Pro 本地备份后，如何从 metadata.json 看懂一条节点，并与 App 里的图、备注、链接、标签、多图和记忆曲线逐项对上号。

一、备份字段与界面对照

  • id：节点唯一标识。
  • timestamp / createdDate：记录时间。
  • imagePath：图片文件名；多图用 | 连接。
  • note：备注正文（可含链接）。
  • tags：标签数组。
  • memoryCurveItems：记忆曲线进度（按节点 id 关联）。

二、三类节点样例

1）视频/知识向

常见结构是：多图 + 摘要备注 + 来源链接 + 主题标签。回到详情页可直接点击链接回源。

2）文章/深度稿

常见结构是：关键信息截图 + 观点备注 + 原文链接 + 分类标签。

3）生活/灵感类

常见结构是：多图 + 当下感受 + 自定义标签，后续按时间线或按标签都能回看。

三、从操作到备份的顺序

  1. 主页新建节点或分享扩展收集素材。
  2. 补备注与链接、添加标签。
  3. 按需加入记忆曲线。
  4. Pro 导出备份，得到 metadata.json 与图片文件。

四、小结

一条节点在界面里是「图 + 文 + 标签 + 时间」，在备份里是结构化字段。理解字段后，你就能在换机、归档和验数时更有把握。
